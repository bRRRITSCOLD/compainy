# `ai` Dev-Team Plugin Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Build a public Claude Code plugin `bRRRITSCOLD/ai` shipping a 4-agent dev team (UX designer, frontend, backend, staff) + shared engineering-principle skills + domain skills + Figma Dev Mode MCP + SessionStart hooks + a handoff skill, distributed via a self-marketplace that also lists `superpowers`.

**Architecture:** Single plugin at repo root. `.claude-plugin/plugin.json` is the manifest (declares the Figma MCP server). `.claude-plugin/marketplace.json` aggregates `ai` (`source: "./"`) + `superpowers` (git source). Agents reference reusable skills by name; principle skills are authored once and shared. Hooks inject session-hygiene guidance and surface handoff files.

**Tech Stack:** Claude Code plugin format (JSON manifests, Markdown agents/skills, bash hooks), Figma Dev Mode MCP (HTTP), `gh` + git.

## Global Constraints

- Repo: `github.com/bRRRITSCOLD/ai`, **public**, owner `bRRRITSCOLD`, git protocol **ssh**.
- Author identity for commits: `Blaine Richardson <blainerrichardson@gmail.com>`.
- Plugin name: **`ai`**. Plugin schema: `.claude-plugin/plugin.json` with `name`, `version` (`0.1.0`), `description`, `author`, `homepage`, `repository`, `license` (`MIT`), `keywords`.
- Every `agents/*.md` frontmatter: `name`, `description`, `model: inherit`, `color`, `tools` (array). Body documents role, "When to invoke", and a `## Operates by` section listing the skills it MUST follow.
- Every `skills/<name>/SKILL.md` frontmatter: `name` (matches dir), `description` (third-person, trigger-rich, ≤ ~500 chars). Body: purpose + a `## Checklist` or `## Process` the agent follows.
- Superpowers is a **required companion**, surfaced via `marketplace.json`; never vendored.
- Principle skills are the single source of truth — agents reference them, never restate them (DRY).
- Pragmatic stance is mandatory copy in the two "principles" skills (verbatim guardrail lines below). KISS/YAGNI break ties.
- Commit message footer on every commit: `Co-Authored-By: Claude Opus 4.8 (1M context) <noreply@anthropic.com>`.

- [ ] **Step 1: Write `skills/handoff/scripts/write-handoff.sh`**

```bash
#!/usr/bin/env bash
# Usage: write-handoff.sh <body-file>
# Prints the path of the handoff file it created.
set -euo pipefail
slug="$(pwd | sed 's#^/##; s#/#-#g')"
dir="${TMPDIR:-/tmp}/claude-handoffs/${slug}"
mkdir -p "$dir"
ts="$(date -u +%Y-%m-%dT%H-%M-%SZ)"
out="${dir}/${ts}.md"
cp "${1:?body file required}" "$out"
echo "$out"
```

### Task 4: SessionStart hook (hygiene nudge + handoff surfacing)

**Files:**
- Create: `hooks/hooks.json`
- Create: `hooks/scripts/session-start.sh`

**Interfaces:**
- Consumes: handoff dir convention from Task 3.
- Produces: a SessionStart hook emitting `additionalContext` to stdout (JSON) so Claude sees the nudge + any recent handoff.

- [ ] **Step 1: Write `hooks/scripts/session-start.sh`**

```bash
#!/usr/bin/env bash
set -euo pipefail
slug="$(pwd | sed 's#^/##; s#/#-#g')"
dir="${TMPDIR:-/tmp}/claude-handoffs/${slug}"
nudge="Session hygiene: between chunks of work, run the handoff skill and start a FRESH session. Small context = sharp Claude."
recent=""
if [ -d "$dir" ]; then
  latest="$(ls -1t "$dir" 2>/dev/null | head -1 || true)"
  [ -n "$latest" ] && recent="A recent handoff exists at ${dir}/${latest} — offer to read it before continuing."
fi
ctx="$nudge"
[ -n "$recent" ] && ctx="$ctx"$'\n'"$recent"
printf '{"hookSpecificOutput":{"hookEventName":"SessionStart","additionalContext":%s}}\n' \
  "$(printf '%s' "$ctx" | python3 -c 'import json,sys;print(json.dumps(sys.stdin.read()))')"
```

- [ ] **Step 2: Make executable + run it**

Run: `chmod +x hooks/scripts/session-start.sh && hooks/scripts/session-start.sh | python3 -c "import json,sys;print(json.load(sys.stdin)['hookSpecificOutput']['additionalContext'][:20])"`
Expected: prints `Session hygiene: bet`

- [ ] **Step 3: Write `hooks/hooks.json`**

```json
{
  "description": "Session hygiene + handoff surfacing for the ai dev-team plugin",
  "hooks": {
    "SessionStart": [

          {
            "type": "command",
            "command": "${CLAUDE_PLUGIN_ROOT}/hooks/scripts/session-start.sh"
          }
        ]
      }
    ]
  }
}
```
